In last few weeks I helped users streppuiu and dominic7il making/using Apache web server and MySQL database server. At the end I put everything in a nice LAMP addon package which includes webserver with php and database server. I also included phpMyAdmin tool to make administration of MySQL databases.

I could run custom web application (use by my friend) and also ownCloud open source file sync and share solution.

Download location: drive.google.com/folderview?id=0B4FdVu0-fJt6V2JwelNzdkc0YkE&usp=sharing#list

mirror1.openelec.tv/lamp/

Install manually from xbmc.

Controlling with systemctl:
systemctl start/stop/restart lamp
systemctl start/stop/restart httpd
systemctl start/stop/restart mysqld

systemctl start/stop/restart lamp.service
systemctl start/stop/restart httpd.service
systemctl start/stop/restart mysqld.service

systemctl start/stop/restart service.web.lamp.service
or with traditional shell scripts:
lamp.start
lamp.stop 

httpd.start 
httpd.stop

mysqld.start 
mysqld.stop

Some required data:

mysql server
username: root
password: 123

http port: 4080
https port: 443
web server root folder
/storage/.kodi/userdata/addon_data/service.web.lamp/www/htdocs/

http://OE_BOX_IP:4080/tests.php
http://OE_BOX_IP:4080/phpinfo.php
http://OE_BOX_IP:4080/db_test.php
http://OE_BOX_IP:4080/exif_test.php
http://OE_BOX_IP:4080/mail_test.php
http://OE_BOX_IP:4080/phpMyAdmin/index.php

to install owncloud
owncloud.org/install/#instructions-server
unpack owncloud-.tar.bz2 to web root folder
tar xjf owncloud-8.0.4.tar.bz2 -C /storage/.kodi/userdata/addon_data/service.web.lamp/www/htdocs/

and open in web browser
http://OE_BOX_IP:4080/owncloud/index.php

fill with
username: admin
password: pass
data folder: leave default /storage/.kodi/userdata/addon_data/service.web.lamp/www/htdocs/owncloud/data
mysql username: root
mysql password: 123
mysql name: owncloud
mysql host: localhost

To install ocDownloader
You must have ownCloud installed (ocDownloader is in addition to ownCloud)
download from github.com/DjazzLab/ocdownloader
wget -O ocDownloader.zip github.com/DjazzLab/ocdownloader/archive/master.zip
unzip ocDownloader.zip -d /storage/.kodi/userdata/addon_data/service.web.lamp/www/htdocs/owncloud/apps
mv /storage/.kodi/userdata/addon_data/service.web.lamp/www/htdocs/owncloud/apps/ocdownloader-master /storage/.kodi/userdata/addon_data/service.web.lamp/www/htdocs/owncloud/apps/ocdownloader
systemctl restart httpd
Enable it in ownCloud applications

To setup sending mail function with php mail() function over msmtp program you need to edit config file msmtprc with smtp server data. After that restart httpd and it should work.
msmtprc sample

If anyone has some special request/interest let me know and I will add some more functionality (like php extensions).

Sources: github.com/vpeter4/OpenELEC.tv/tree/lamp

To disable mysql service permanently and use only web server
systemctl stop mysqld.service
systemctl disable mysqld.service
or to disable web server and use only mysql server
systemctl stop httpd.service
systemctl disable httpd.service

Anyway as promised earlier here's a writeup of things I've changed/fixed in regards to apache:

In httpd.conf add the following to the list of modules, otherwise basic authentication will not work:
LoadModule authn_core_module mod_authn_core.so
As a result of the above change you'll also need to change:
AllowOverride None
To:
AllowOverride All
This will stop issues with http error handling when using basic authentication.

Also if you prefer full Apache directory listing, un-comment the below in httpd.conf:
Include /storage/.xbmc/userdata/addon_data/service.web.lamp/srvroot/conf/extra/httpd-autoindex.conf

If you'd like to show a basic server signature on your directory listings, add the below to httpd.conf:
ServerTokens Full
ServerSignature On

To add automatic mime type based descriptions your your directory listings as well as putting folders on top, add the below to httpd.conf:
IndexOptions IgnoreCase FancyIndexing FoldersFirst
AddDescription "MPEG Layer 3 Format" .mp3
AddDescription "GZIP compressed TAR archive" .tgz .tar.gz
AddDescription "GZIP compressed archive" .Z .z .gz .zip
AddDescription "RAR compressed archive" .rar
AddDescription "TAR compressed archive" .tar
AddDescription "ZIP compressed archive" .zip
AddDescription "Windows executable file" .exe
AddDescription "Common Gateway Interface" .cgi
AddDescription "Joint Photographics Experts Group" .jpg .jpeg .jpe
AddDescription "Graphic Interchange Format" .gif
AddDescription "Portable Network Graphic" .png
AddDescription "Vector graphic" .ps .ai .eps
AddDescription "Hypertext Markup Language" .html .shtml .htm
AddDescription "Cascading Style Sheet" .css
AddDescription "DocType Definition" .dtd
AddDescription "Extensible Markup Language" .xml
AddDescription "Win32 compressed HTML help" .chm
AddDescription "Adobe Portable Document Format" .pdf
AddDescription "Plain text file" .txt .nfo .faq .readme
AddDescription "Unix man page" .man
AddDescription "Email data" .eml .mbox
AddDescription "Microsoft Word document" .doc
AddDescription "PHP: Hypertext Preprocessor script" .php  .php3 .php4
AddDescription "PHP: Hypertext Preprocessor source code" .phps
AddDescription "Javascript" .js
AddDescription "Java code"  .java
AddDescription "Unix shell script" .sh .shar .csh .ksh .command
AddDescription "Mac OS X shell script" .command
AddDescription "Configuration file" .conf
AddDescription "Mac OS X terminal" .term
AddDescription "BitTorrent file" .torrent
AddDescription "Windows link" .lnk .url

For the full directory listings, you'll need to add the apache icons folder into the build, as it's currently missing. You can download it from here:
drive.google.com/file/d/0B9tjMpKj5S4hTDh2V0dUMDJ4Nlk/edit?usp=sharing

You'll also need to set the path to the icon folder in extra/httpd-autoindex.conf.

I also noticed one strange thing with MySQL - by default it seems to allow root to login without a password from any host. Then there's specific entries for root on localhost,127.0.0.1 and openelec (hostname) which do require a password. This would mean in theory if the MySQL port was publically visible anyone could log in as root without a password. Ofcourse easily correctable from command line or even phpMyAdmin; just might be an idea to override that as default behavior.

Hope this will be of some use.
